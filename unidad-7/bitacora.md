## 👌Actividad 02: Exploración de Matter.js

Explica con tus palabras qué hace cada uno de esos conceptos
### Engine
El Engine es el cerebro de la simulación.
Se encarga de calcular todo: gravedad, colisiones, movimiento.
Actualiza el estado de los objetos en cada frame.
como el “motor físico” que hace que todo se mueva.

### World
El World es el contenedor donde vive todo.
Ahí se guardan todos los objetos físicos.
Incluye propiedades globales como la gravedad.
Es básicamente el “escenario” de la simulación.

### Bodies
Los Bodies son los objetos físicos.
Pueden ser cajas, círculos o formas más complejas.
Tienen propiedades como masa, velocidad, fricción, rebote.
Si estamos hablando de animar una palabra, cada letra podría ser un Body.

### Constraint
Un Constraint es una conexión entre cuerpos.
Puede comportarse como una cuerda, resorte o unión rígida.
Sirve para mantener relaciones entre objetos.
Por ejemplo, puedes unir letras para que se muevan juntas como si fueran una palabra flexible.

### MouseConstraint
El MouseConstraint conecta el mouse con la simulación.
Permite hacer clic y arrastrar objetos.
Simula que “agarras” un cuerpo con el cursor.
Es lo que hace la interacción: mover letras con el mouse, lanzarlas, etc.
