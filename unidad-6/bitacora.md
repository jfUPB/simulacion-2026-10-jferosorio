Reconozco una composició intensa, estructurada y vibrante que la anterior. Aquí las decisiones visuales parecen orientadas hacia la energía urbana, la organización de datos y el movimiento mecánico.

### Composición

La obra ocupa casi todo el plano con formas curvas y bandas fragmentadas. No hay grandes vacíos; la superficie está activa de borde a borde. Las estructuras principales parecen nacer desde distintos centros y expandirse en arcos, diagonales y ondas. Esto genera una composición saturada pero organizada, donde cada sector tiene peso visual.

### Densidad

La densidad es alta. Hay numerosos módulos pequeños distribuidos en toda la imagen: rectángulos, bloques y segmentos repetidos. Algunas zonas concentran aún más elementos, creando focos de intensidad visual. La alta densidad transmite complejidad, abundancia de información y dinamismo constante.

### Dirección del movimiento

Predominan trayectorias curvas: espirales, abanicos, ondas horizontales y arcos radiales. Muchas líneas parecen girar alrededor de centros invisibles, mientras otras fluyen lateralmente. Esto crea una sensación de circulación, expansión y velocidad. El ojo no se queda quieto: recorre rutas múltiples.

### Color

La paleta es potente y contrastada: azules intensos, naranjas, amarillos, rojos, marrones, negros y blancos. Los colores cálidos aportan energía y foco, mientras los azules equilibran y enfrían ciertas áreas. El contraste entre tonos vivos y neutros genera profundidad y jerarquía.

### Ritmo

El ritmo visual es rápido y sincopado. Surge de la repetición de pequeños módulos rectangulares colocados en secuencias. Algunas franjas tienen pulsaciones regulares; otras se interrumpen y cambian de tamaño. Esto recuerda patrones musicales, tráfico urbano o datos digitales en movimiento.

### Repetición y variación

Se repite un vocabulario claro: bloques rectangulares organizados en bandas curvas. Sin embargo, cambian tamaño, color, espaciado, grosor, orientación y densidad. Esa estrategia mantiene cohesión general sin caer en monotonía. Todo parece pertenecer a un mismo sistema modular.


### ¿Por qué estas decisiones me parecen potentes?

Porque convierten formas geométricas simples en una imagen llena de energía y complejidad. La obra parece simultáneamente matemática y emocional: hay estructura rigurosa, pero también impulso y caos controlado. Sugiere mapas, circuitos, ciudades, música visualizada o flujos de información. Tiene capacidad de atrapar la mirada durante mucho tiempo porque siempre aparecen nuevos detalles.



### Hipótesis: reglas o sistema detrás de la pieza

1. **Definir varios centros o ejes de curvatura** en el plano.
2. **Trazar bandas circulares, onduladas o radiales** desde esos ejes.

Reconozco una serie de decisiones visuales  intencionales que convierten una composición aparentemente simple en una imagen dinámica y expresiva.

### Composición

La pieza distribuye varios grupos de líneas curvas en el espacio vertical del formato. No están centrados ni simétricos; más bien aparecen dispersos de forma equilibrada, generando zonas de vacío amplias entre ellos. Esto hace que cada conjunto respire y que la mirada recorra toda la superficie. La composición se siente orgánica, como si las formas crecieran desde distintos puntos del plano.

### Densidad

La densidad varía claramente: algunas zonas concentran muchas líneas superpuestas, mientras otras apenas contienen unas pocas marcas suaves. Esa alternancia entre concentración y vacío crea jerarquía visual. Las áreas más densas atraen la atención y las más ligeras permiten descanso visual.

### Dirección del movimiento

Predominan líneas ascendentes, onduladas y curvadas hacia arriba o en diagonal. Muchas parecen brotar desde abajo y expandirse, como llamas, humo, plantas marinas o corrientes de aire. Esto genera sensación de crecimiento, elevación y flujo continuo. No hay rigidez; todo parece desplazarse.

### Color

Se usan principalmente tres colores suaves: azul, rojo/coral y amarillo. El azul domina la base de muchas formas, mientras el rojo y amarillo aparecen como acentos. La paleta es ligera y luminosa. El contraste cromático no es agresivo, sino delicado, lo que refuerza una atmósfera etérea.

### Ritmo

El ritmo surge de la repetición de trazos finos paralelos que se curvan de manera semejante. Los grupos parecen pulsaciones visuales: aparecen, se expanden y desaparecen. El ojo salta de un racimo a otro siguiendo cadencias suaves, casi musicales.

### Repetición y variación

Se repite un mismo lenguaje formal: líneas largas, fluidas, curvadas y agrupadas. Pero cada grupo cambia en tamaño, orientación, densidad y mezcla de color. Esa combinación evita monotonía y mantiene unidad visual. Todo pertenece al mismo sistema, aunque nada es idéntico.


### ¿Por qué estas decisiones me parecen potentes?

Porque logran transmitir energía y movimiento usando elementos mínimos: solo líneas, color y espacio. No necesitan figuras reconocibles para sugerir naturaleza, viento, fuego o crecimiento. Además, la tensión entre control (patrones repetidos) y espontaneidad (curvas libres) vuelve la imagen viva. Parece calculada y libre al mismo tiempo.


### Hipótesis: reglas o sistema detrás de la pieza

Creo que la obra podría surgir de un sistema generativo o de reglas simples como estas:

1. **Crear puntos de origen aleatorios** distribuidos en el plano.
2. **Emitir múltiples líneas desde cada punto** con pequeñas variaciones de ángulo.

## 👌Actividad 02: Agentes autónomos y steering forces
Antes de entrar a ejemplos concretos, vas a aclarar tres ideas fundamentales:

¿Qué es un agente autónomo?
¿Qué es una steering force?
¿En qué se diferencia una steering force de fuerzas como gravedad, viento o fricción?



## ¿Qué es un agente autónomo?
es una entidad dentro de una simulación o sistema que **percibe su entorno, toma decisiones y actúa por sí misma** para cumplir objetivos,
En videojuegos, animación o IA, puede ser: Un NPC que patrulla y persigue enemigos o n pez que nada evitando obstáculos.

Tiene normalmente tres partes:
1. **Percepción**: detecta entorno (paredes, enemigos, comida, aliados).
2. **Decisión**: elige qué hacer.
3. **Acción/movimiento**: se desplaza o actúa.

es un personaje con cierta voluntad programada


## ¿Qué es una steering force?

fuerza de guiado o conducción, es una fuerza calculada que **modifica la dirección y velocidad del agente para producir comportamientos inteligentes de movimiento**
No busca simular física realista solamente, sino comportamiento.

Ejemplos clásicos:
* **Seek**: ir hacia un objetivo.
* **Flee**: huir de algo.
* **Arrive**: llegar suavemente frenando.


Normalmente se suma como vector:


## ¿En qué se diferencia una steering force de gravedad, viento o fricción?

### 1. Propósito

### Fuerzas físicas reales:

Buscan representar leyes naturales.

* Gravedad atrae masas.
* Fricción desacelera por contacto.
* Viento empuja por fluidos.

### Steering force:

Busca producir **comportamiento inteligente o intencional**.

* “quiero ir allá”
* “quiero escapar”
* “quiero no chocar”
* “quiero seguir al grupo”


### 2. Origen

### Fuerzas físicas:

Provienen del mundo físico.

### Steering force:

Proviene del algoritmo de decisión del agente.
Es una fuerza “artificial” 

### 3. Naturaleza

### Fuerzas físicas:

Generalmente siguen ecuaciones científicas.
Ejemplo:
F=ma

### Steering force:

Usa vectores diseñados por el programador.
Ejemplo seek:
\vec{F}*{steer}=\vec{v}*{desired}-\vec{v}_{actual}



## Ejemplo simple

Un enemigo ve al jugador.

* gravedad = lo atraería hacia abajo.
* fricción = lo frenaría.
* viento = lo empujaría lateralmente.
* steering seek = corre hacia el jugador.
* steering avoid = rodea una pared.



**La física mueve cuerpos; las steering forces mueven intenciones.**

## 👌Actividad 03: Flow Fields

## ¿Cómo está construido el campo de flujo?

Se construye como una **rejilla 2D** dividida en columnas y filas. Cada celda guarda un **vector de dirección**. La resolución define el tamaño de cada celda (ej. 20 px). Los vectores pueden generarse aleatoriamente o con **Perlin noise** para crear continuidad suave. ([Nature of Code][1])

## ¿Qué representa cada celda o vector del campo?

Cada celda representa una **dirección ideal de movimiento** en esa zona del espacio. Es como una flecha invisible que le dice al agente hacia dónde avanzar. ([Nature of Code][1])

## ¿Cómo usa un agente su posición para consultar el campo?

El agente toma su posición `(x,y)` y la convierte en índices de la rejilla:

```text id="h9m3c1"
col = floor(x / resolution)
row = floor(y / resolution)
```

Así identifica en qué celda está y consulta el vector almacenado allí. ([Nature of Code][1])

## ¿Cómo se convierte el vector consultado en una decisión de movimiento?

El vector consultado se escala a `maxspeed` (velocidad deseada). Luego se compara con la velocidad actual:

\vec F_{steer}=\vec v_{desired}-\vec v_{actual}

Esa steering force se limita con `maxforce` y se aplica al agente como aceleración. ([Nature of Code][1])

---

## Parámetros importantes del sistema

* **resolution**: tamaño de cada celda.
* **maxspeed**: velocidad máxima del agente.
* **maxforce**: capacidad de giro/cambio de dirección.
* **cantidad de agentes**: densidad visual y complejidad emergente.
* **noise scale / seed**: forma general del flujo si usas Perlin noise. ([Nature of Code][1])

---

## 👌Actividad 04: Flocking

<img width="654" height="246" alt="image" src="https://github.com/user-attachments/assets/e165ec35-101e-4b61-abb1-9ec18280334f" />


## Explica con tus palabras las tres reglas básicas

### Separación

Cada agente evita acercarse demasiado a sus vecinos. Si otro boid invade su espacio personal, se aleja para prevenir choques y saturación.

### Alineación

Cada agente intenta igualar su dirección y velocidad con los vecinos cercanos. Esto hace que el grupo avance coordinado.

### Cohesión

Cada agente busca acercarse al centro promedio del grupo local. Sirve para mantener la bandada unida y no fragmentada. ([Nature of Code][1])

---

## ¿Qué parámetros controlan estas reglas?

* **Separation weight**: fuerza con la que evita vecinos.
* **Alignment weight**: cuánto copia dirección grupal.
* **Cohesion weight**: cuánto busca reunirse.
* **Neighbor distance**: radio para detectar vecinos.
* **Desired separation**: distancia mínima deseada.
* **maxspeed**: velocidad máxima.
* **maxforce**: capacidad de giro / reacción.
* **Cantidad de agentes**: densidad y complejidad colectiva. ([Nature of Code][1])

---

## Modificación de pesos y efecto visual

### Modificación:

* Separation = **2.0**
* Alignment = **1.0**
* Cohesion = **0.6**

### Efecto:

El grupo mantiene distancia entre individuos, se forman bandadas más abiertas y elegantes. Menos choques, más respiración visual. La masa se mueve como aves reales en migración.

### Otra prueba:

* Separation baja + Cohesion alta

### Efecto:

El grupo colapsa hacia racimos densos, con choques y movimientos tensos.

---

## Comportamiento emergente observado

Con la primera configuración:

* **Compacto:** medio
* **Disperso:** moderado
* **Estable:** alto
* **Nervioso:** bajo
* **Caótico:** bajo
* **Fluido:** alto

El flock parece una inteligencia colectiva suave que se reorganiza constantemente.


## ¿Qué atmósfera visual produce el flocking?

Produce sensación de:
* Vida orgánica
* Naturaleza coordinada, manadas bandadas de pajaros, rebaños
* Migración / libertad
* Ecosistema inteligente
También puede sentirse hipnótico cuando hay muchos agentes.



## ¿En qué tipo de relación con una canción podría funcionar mejor este algoritmo?
Funciona excelente como **visualizador reactivo** a música con estructura rítmica y evolución progresiva:

* Ambient
* Downtempo
* House melódico
* Drum & Bass líquido
* Cinemática / épica
* Synthwave espacial

### Relación ideal música-movimiento:

* **Beat fuerte** → aumenta separación / velocidad
* **Melodía suave** → sube cohesión
* **Drop** → caos temporal y expansión
* **Breakdown** → flock lento y flotante

Así el enjambre “baila” con la canción.

## Bitácora de aplicación 

# Concepto visual
Me encanda el mar y su comportamiento, busque materializar esta naturaleza en mi obre prodcurando que la pieza traduce la fluidez y el ciclo del ecosistema marino en un sistema de flocking y campos de corriente (flow fields). No se trata solo de mover peces, sino de evocar la calma del mar, la energía del verano, la quietud del invierno y la fragilidad del deshielo. Los bancos de peces representan la vida que se adapta a las condiciones del océano, y el color cambiante simboliza la vitalidad y la conexión con la música. Se busca tambien incorporar siempre un elemento educativo, algun tema a explicar o presentar, el experimento sirve como un recurso visual para esto, en este caso explicar el comportamiento animal marino.

<br>

# Relación con el tema musical
Elegi Utu Bang Banga, ua cancion en estilo regge para iniciar la obra, ya que lo primero que pensamos cuando nos mencionan el mar, es la playa, la vida tropical y la tranquilidad La canción se encuentra aprox≈140 BPM y compás 4/4 no es un fondo decorativo: su ritmo gobierna el cambio de color de los peces , su amplitud modula la luminosidad del fondo y la cantidad de espuma, y su energía general influye en la velocidad del cambio cromático. 

<br>

# Reactividad al audio
El sistema reacciona en tiempo real a tres características del audio:

Amplitud → modula el brillo del fondo (más volumen, más luz) y la cantidad de partículas de espuma.

Energía en frecuencias medias/graves (60‑150 Hz) → detección de beats para cambiar el color arcoíris de los peces de forma sincronizada con el ritmo.

Espectro general → influye en la velocidad del cambio cromático y en la generación de pequeñas chispas de luz.
El resultado es un diálogo constante entre lo que se oye y lo que se ve.

<br>

# Interacción performativa
El usuario puede “tocar” la obra en vivo:

Mouse → clic: arranca la música y luego recorre cíclicamente las tres corrientes marinas (verano, invierno, deshielo).

Barra espaciadora → alterna entre velocidad normal y velocidad lenta, permitiendo ralentizar el movimiento en cualquier momento.
Ninguna acción es automática; cada gesto modifica el estado emocional de la pieza, como si se tratara de un instrumento.


Interacción con sentido musical
Cada interacción tiene una razón expresiva alineada con la música:

Cambiar de estación con el clic permite adaptar la visual a la energía de la canción (verano = rápido y disperso, invierno = comprimido y pausado).

La barra espaciadora (modo lento) sirve para destacar pasajes más íntimos o silenciosos de la melodía, ofreciendo un respiro visual.
No hay interacciones arbitrarias: todas refuerzan la experiencia de “navegar” por las corrientes marinas al ritmo de la pieza musical.

# Uso justificado del algoritmo
Se eligió flocking porque el movimiento de los peces, para este experimetno y lo que queremos contemplar y plasmar debe parecer natural, emergente y grupal, no individualmente aleatorio. Las reglas de separación, alineamiento y cohesión producen bancos orgánicos.
Se añadió un flow field (campo de vectores basado en ruido Perlin) para simular las corrientes marinas, un fenómeno que el flocking por sí solo no puede generar. La combinación de ambos permite que los peces sigan tanto las tendencias grupales como las fuerzas ambientales externas, logrando la metáfora de “corrientes que afectan la vida marina”.

# Mapa de decisiones

| Decisión de diseño | Intención |
|---|---|
| Colores de fondo en azules (HSB) según estación | Simular el mar y diferenciar visualmente las tres estaciones: verano (azul claro), invierno (azul oscuro profundo), deshielo (cian blanquecino). |
| Flocking con alta cohesión y compresión en invierno | Transmitir sensación de calma, agrupación protectora y movimiento contenido, acorde a la quietud del invierno. |
| Flocking disperso, baja cohesión y ruido errático en verano | Representar energía, caos, dinamismo y movimiento rápido, acorde a la vitalidad del verano. |
| Flocking muy comprimido y extremadamente lento en deshielo | Evocar fragilidad, lentitud, casi quietud, como un ecosistema en estado de transición. |
| Campo de vectores (flow field) basado en ruido Perlin | Crear corrientes marinas orgánicas y continuas, no solo comportamiento grupal; el flocking por sí solo no puede simular fuerzas ambientales externas. |
| Flow field caótico y fuerte en verano | Reforzar la sensación de movimiento errático y energía desbordada. |
| Flow field suave, uniforme y débil en invierno | Producir una corriente lenta y predecible, que invita a la calma. |
| Flow field casi estático en deshielo | Representar la ausencia de corriente, un mar detenido. |
| Detección de beats en frecuencias graves (60-150 Hz) | Sincronizar el cambio de color arcoíris de los peces con el ritmo de la canción (≈140 BPM). |
| Velocidad base de peces: 5.0 (verano), 2.2 (invierno), 1.2 (deshielo) | Adaptar el movimiento de los peces al tempo musical: verano = negras rápidas (140 BPM), invierno = half-time (70 BPM), deshielo = movimiento extremadamente pausado. |
| Estela con alpha = 35 (fondo semi-transparente) | Acentuar el rastro de los peces, dando sensación de continuidad, fluidez y persistencia de la memoria visual. |
| Estela muy suave en versiones anteriores (alpha 10) | Buscar una estela más sutil y etérea (descartado en favor de una estela más marcada por petición del usuario). |
| Espuma de mar: partículas blancas + líneas curvas | Añadir una capa textural superior que refuerce la sensación de superficie marina, espuma y movimiento de las olas. |
| Cantidad de peces: 300 (3 grupos de 100) | Crear densidad visual, sensación de banco masivo y vida abundante, sin saturar el rendimiento. |
| Colores arcoíris de los peces: HSB con saturación y brillo al 100% | Obtener colores vivos, vibrantes y muy saturados que contrasten con el fondo azul y transmitan energía tropical. |
| Cambio de color sincronizado por beat (avanza 30° de tono por cada beat) | Que el color de los peces cambie al ritmo de la música, haciendo visible el pulso de la canción. |
| Interacción con clic del mouse | Permite al usuario cambiar de estación en vivo, interviniendo performativamente sobre las corrientes marinas. |
| Interacción con barra espaciadora | Alterna entre velocidad normal y velocidad lenta, ofreciendo un recurso expresivo para ralentizar la pieza en momentos de calma o silencio musical. |
| Pantalla completa adaptativa | Crear una experiencia inmersiva, sin bordes ni distracciones externas. |
| Sin dashboards ni instrucciones visibles | Mantener la obra como una pieza performática, no como un prototipo de laboratorio; el usuario descubre la interacción por sí mismo. |
| Fondo modulado por amplitud del audio | El volumen de la música afecta el brillo del fondo: más volumen = más luz, vinculando visualmente la energía sonora. |
| Partículas de luz (chispas blancas) con alta energía | Reforzar la atmósfera marina y añadir un detalle estético que responde a los momentos de mayor intensidad musical. |

// -------------------- CONFIGURACIÓN GLOBAL --------------------
let boids = [];
let audio;
let fft;
let amplitud;
let musicaIniciada = false;

let estacion = 0;         // 0: verano, 1: invierno, 2: deshielo
let slowMode = false;

// Parámetros de flocking (velocidades ajustadas al ritmo)
let flockParams = {
    maxSpeed: 3.5,
    maxForce: 0.2,
    cohesionWeight: 0.5,
    alignmentWeight: 0.6,
    separationWeight: 0.8,
    perceptionRadius: 70,
    separationRadius: 40,
    flowFieldWeight: 0.8
};

// Flow field
let flowField = [];
let cols, rows;
let resolution = 40;

// Colores base de cada estación en HSB (azules)
const coloresEstacion = [
    { h: 200, s: 70, b: 80 },   // verano (azul claro)
    { h: 210, s: 80, b: 40 },   // invierno (azul oscuro)
    { h: 180, s: 60, b: 90 }    // deshielo (cian)
];

let hueOffset = 0;
let beatCooldown = 0;
let currentHueStep = 0;

// Espuma: partículas y líneas curvas
let foamParticles = [];
let foamLines = [];

// -------------------- CLASE BOID (PEZ) --------------------
class Boid {
    constructor(x, y) {
        this.position = createVector(x, y);
        this.velocity = p5.Vector.random2D();
        this.velocity.setMag(random(1, flockParams.maxSpeed * 0.8));
        this.acceleration = createVector();
        this.size = random(8, 14);
        this.tailAngle = random(-0.3, 0.3);
    }

    flock(boids) {
        let cohesion = createVector();
        let alignment = createVector();
        let separation = createVector();
        let total = 0;

        for (let other of boids) {
            let d = p5.Vector.dist(this.position, other.position);
            if (other !== this && d < flockParams.perceptionRadius) {
                cohesion.add(other.position);
                alignment.add(other.velocity);
                total++;
                if (d < flockParams.separationRadius) {
                    let diff = p5.Vector.sub(this.position, other.position);
                    diff.div(d);
                    separation.add(diff);
                }
            }
        }
        if (total > 0) {
            cohesion.div(total);
            cohesion.sub(this.position);
            cohesion.setMag(flockParams.maxForce);
            cohesion.mult(flockParams.cohesionWeight);

            alignment.div(total);
            alignment.setMag(flockParams.maxForce);
            alignment.mult(flockParams.alignmentWeight);

            separation.setMag(flockParams.maxForce);
            separation.mult(flockParams.separationWeight);

            this.applyForce(cohesion);
            this.applyForce(alignment);
            this.applyForce(separation);
        }
    }

    applyForce(force) {
        this.acceleration.add(force);
    }

    applyFlowField() {
        let col = floor(this.position.x / resolution);
        let row = floor(this.position.y / resolution);
        col = constrain(col, 0, cols - 1);
        row = constrain(row, 0, rows - 1);
        let flowForce = flowField[col][row].copy();
        flowForce.mult(flockParams.flowFieldWeight);
        this.applyForce(flowForce);
    }

    update() {
        this.velocity.add(this.acceleration);
        let currentMaxSpeed = slowMode ? flockParams.maxSpeed * 0.5 : flockParams.maxSpeed;
        this.velocity.limit(currentMaxSpeed);
        this.position.add(this.velocity);
        this.acceleration.mult(0);

        if (this.position.x < -this.size) this.position.x = width + this.size;
        if (this.position.x > width + this.size) this.position.x = -this.size;
        if (this.position.y < -this.size) this.position.y = height + this.size;
        if (this.position.y > height + this.size) this.position.y = -this.size;
    }

    show() {
        push();
        translate(this.position.x, this.position.y);
        let angle = this.velocity.heading();
        rotate(angle);

        // Color arcoíris vibrante, sincronizado con el ritmo (cambia en cada beat)
        let hueVal = (hueOffset + currentHueStep * 30) % 360;
        fill(hueVal, 100, 100);
        noStroke();

        beginShape();
        vertex(this.size, 0);
        vertex(-this.size * 0.6, -this.size * 0.5);
        vertex(-this.size * 0.3, 0);
        vertex(-this.size * 0.6, this.size * 0.5);
        endShape(CLOSE);

        stroke(hueVal, 100, 90);
        strokeWeight(1.5);
        line(-this.size * 0.5, 0, -this.size * 0.9, -this.size * 0.4 * (1 + this.tailAngle));
        line(-this.size * 0.5, 0, -this.size * 0.9, this.size * 0.4 * (1 + this.tailAngle));
        pop();
    }
}

// -------------------- ESPUMA: PARTÍCULAS BLANCAS --------------------
class FoamParticle {
    constructor(x, y) {
        this.x = x;
        this.y = y;
        this.size = random(2, 5);
        this.life = 255;
        this.vx = random(-0.3, 0.3);
        this.vy = random(-0.5, -0.1);
    }
    update() {
        this.x += this.vx;
        this.y += this.vy;
        this.life -= 4;
    }
    show() {
        noStroke();
        fill(0, 0, 100, this.life);
        ellipse(this.x, this.y, this.size);
    }
    isDead() {
        return this.life <= 0;
    }
}

// -------------------- ESPUMA: LÍNEAS CURVAS BLANCAS --------------------
class FoamLine {
    constructor() {
        this.x = random(width);
        this.y = random(height);
        this.w = random(30, 80);
        this.h = random(5, 15);
        this.life = 200;
        this.angle = random(TWO_PI);
    }
    update() {
        this.life -= 2;
    }
    show() {
        push();
        translate(this.x, this.y);
        rotate(this.angle);
        noFill();
        stroke(0, 0, 100, this.life * 0.4);
        strokeWeight(random(1, 2.5));
        beginShape();
        for (let t = -this.w/2; t <= this.w/2; t += 5) {
            let offsetY = sin(t * 0.2) * this.h;
            vertex(t, offsetY);
        }
        endShape();
        pop();
    }
    isDead() {
        return this.life <= 0;
    }
}

// -------------------- FLOW FIELD (CORRIENTES MARINAS) --------------------
function actualizarFlowField() {
    let noiseScale, noiseStrength, angleOffset;
    switch(estacion) {
        case 0: // Verano: caótico, rápido
            noiseScale = 0.012;
            noiseStrength = 3.0;
            angleOffset = 0;
            break;
        case 1: // Invierno: uniforme y lento (half-time)
            noiseScale = 0.003;
            noiseStrength = 1.0;
            angleOffset = PI / 4;
            break;
        case 2: // Deshielo: muy suave, casi estático
            noiseScale = 0.001;
            noiseStrength = 0.6;
            angleOffset = PI / 6;
            break;
    }
    
    for (let i = 0; i < cols; i++) {
        for (let j = 0; j < rows; j++) {
            let x = i * resolution;
            let y = j * resolution;
            let angle = noise(x * noiseScale, y * noiseScale) * TWO_PI * noiseStrength + angleOffset;
            if (estacion === 0) angle += random(-0.6, 0.6);
            flowField[i][j] = p5.Vector.fromAngle(angle);
            let mag = (estacion === 0) ? 0.7 : (estacion === 1 ? 0.25 : 0.12);
            flowField[i][j].setMag(mag);
        }
    }
}

// -------------------- PECES: 3 grupos de 100 (total 300) --------------------
function crearTodosLosPeces() {
    let cantidades = [100, 100, 100];
    for (let g = 0; g < 3; g++) {
        let centerX = random(width * 0.2, width * 0.8);
        let centerY = random(height * 0.2, height * 0.8);
        let cantidad = cantidades[g];
        for (let i = 0; i < cantidad; i++) {
            let offsetX = random(-120, 120);
            let offsetY = random(-120, 120);
            boids.push(new Boid(centerX + offsetX, centerY + offsetY));
        }
    }
}

// -------------------- DETECCIÓN DE BEATS (140 BPM) --------------------
function detectarBeat() {
    if (!musicaIniciada) return false;
    if (beatCooldown > 0) {
        beatCooldown--;
        return false;
    }
    let energiaRitmo = fft.getEnergy(60, 150);
    let nivelGeneral = amplitud.getLevel();
    let umbral = 180 + nivelGeneral * 50;
    if (energiaRitmo > umbral) {
        beatCooldown = 15;
        return true;
    }
    return false;
}

// -------------------- AJUSTE RÍTMICO DE VELOCIDADES (140 BPM / 70 BPM half-time) --------------------
function actualizarParametrosEstacion() {
    switch(estacion) {
        case 0: // Verano: rápido, negras a 140 BPM
            flockParams.maxSpeed = 5.0;
            flockParams.maxForce = 0.35;
            flockParams.cohesionWeight = 0.25;
            flockParams.alignmentWeight = 0.5;
            flockParams.separationWeight = 0.9;
            flockParams.perceptionRadius = 85;
            flockParams.separationRadius = 50;
            flockParams.flowFieldWeight = 1.1;
            break;
        case 1: // Invierno: half-time (70 BPM), más lento y comprimido
            flockParams.maxSpeed = 2.2;
            flockParams.maxForce = 0.18;
            flockParams.cohesionWeight = 0.85;
            flockParams.alignmentWeight = 0.7;
            flockParams.separationWeight = 0.5;
            flockParams.perceptionRadius = 65;
            flockParams.separationRadius = 35;
            flockParams.flowFieldWeight = 0.55;
            break;
        case 2: // Deshielo: muy lento, redondas (sensación de calma extrema)
            flockParams.maxSpeed = 1.2;
            flockParams.maxForce = 0.12;
            flockParams.cohesionWeight = 1.1;
            flockParams.alignmentWeight = 0.8;
            flockParams.separationWeight = 0.4;
            flockParams.perceptionRadius = 60;
            flockParams.separationRadius = 30;
            flockParams.flowFieldWeight = 0.35;
            break;
    }
    for (let b of boids) {
        if (b.velocity.mag() > flockParams.maxSpeed) {
            b.velocity.setMag(flockParams.maxSpeed * 0.9);
        }
    }
}

// -------------------- INTERACCIONES --------------------
function mousePressed() {
    if (!musicaIniciada) {
        userStartAudio().then(() => {
            audio.loop();
            musicaIniciada = true;
            crearTodosLosPeces();
        }).catch(err => console.log("Error al iniciar audio:", err));
        return;
    }
    estacion = (estacion + 1) % 3;
    actualizarParametrosEstacion();
    actualizarFlowField();
}

function keyPressed() {
    if (key === ' ') {
        slowMode = !slowMode;
        return false;
    }
}

// -------------------- GENERAR ESPUMA --------------------
function generarEspuma() {
    let nivelAudio = musicaIniciada ? amplitud.getLevel() : 0.2;
    let numParticles = floor(map(nivelAudio, 0.05, 0.7, 2, 8));
    for (let i = 0; i < numParticles; i++) {
        foamParticles.push(new FoamParticle(random(width), random(height)));
    }
    if (random() < 0.04) {
        foamLines.push(new FoamLine());
    }
    if (foamParticles.length > 500) foamParticles.splice(0, 150);
    if (foamLines.length > 150) foamLines.splice(0, 50);
}

// -------------------- SETUP --------------------
function setup() {
    createCanvas(windowWidth, windowHeight);
    colorMode(HSB, 360, 100, 100);
    audio = loadSound('music.mp3', 
        () => console.log('Audio cargado'),
        (err) => console.error('Error al cargar audio:', err)
    );
    fft = new p5.FFT(0.8, 256);
    fft.setInput(audio);
    amplitud = new p5.Amplitude();
    amplitud.setInput(audio);
    
    cols = floor(width / resolution) + 1;
    rows = floor(height / resolution) + 1;
    flowField = Array(cols);
    for (let i = 0; i < cols; i++) {
        flowField[i] = Array(rows);
        for (let j = 0; j < rows; j++) flowField[i][j] = createVector(0, 0);
    }
    actualizarFlowField();
    actualizarParametrosEstacion();
    background(200, 70, 80);
}

// -------------------- DRAW --------------------
function draw() {
    let nivelAudio = musicaIniciada ? amplitud.getLevel() : 0.2;
    nivelAudio = constrain(nivelAudio, 0.05, 0.8);
    
    // Fondo con estela MÁS MARCADA (alpha 35 en lugar de 10)
    let base = coloresEstacion[estacion];
    let brilloFactor = map(nivelAudio, 0.05, 0.7, 0.8, 1.2);
    brilloFactor = constrain(brilloFactor, 0.6, 1.3);
    let h = base.h;
    let s = constrain(base.s * (0.9 + brilloFactor * 0.1), 0, 100);
    let bri = constrain(base.b * (0.8 + brilloFactor * 0.2), 0, 100);
    fill(h, s, bri, 35);  // alpha 35: estela más visible y duradera
    noStroke();
    rect(0, 0, width, height);
    
    // Sincronización rítmica del color arcoíris
    if (detectarBeat()) {
        currentHueStep = (currentHueStep + 1) % 12;
        hueOffset = (hueOffset + 30) % 360;
    }
    hueOffset = (hueOffset + nivelAudio * 1.5) % 360;
    
    // Actualizar y dibujar peces
    for (let boid of boids) {
        boid.flock(boids);
        boid.applyFlowField();
        boid.update();
        boid.show();
    }
    
    // Capa de espuma (blanca, sobre todo)
    generarEspuma();
    for (let i = foamParticles.length-1; i >= 0; i--) {
        foamParticles[i].update();
        foamParticles[i].show();
        if (foamParticles[i].isDead()) foamParticles.splice(i,1);
    }
    for (let i = foamLines.length-1; i >= 0; i--) {
        foamLines[i].update();
        foamLines[i].show();
        if (foamLines[i].isDead()) foamLines.splice(i,1);
    }
    
    // Pequeñas chispas de luz
    if (nivelAudio > 0.3 && random() < 0.03) {
        push();
        noStroke();
        fill(0, 0, 100, 60);
        ellipse(random(width), random(height), random(3, 8));
        pop();
    }
}

function windowResized() {
    resizeCanvas(windowWidth, windowHeight);
    cols = floor(width / resolution) + 1;
    rows = floor(height / resolution) + 1;
    flowField = Array(cols);
    for (let i = 0; i < cols; i++) {
        flowField[i] = Array(rows);
        for (let j = 0; j < rows; j++) flowField[i][j] = createVector(0, 0);
    }
    actualizarFlowField();
}
